
# Decisions

Store ADR-style decisions here.

Recommended format for each decision file:

- `Status`: proposed, accepted, superseded, or rejected.
- `Context`: what pressure or constraint forced the decision.
- `Decision`: the chosen option.
- `Consequences`: tradeoffs, follow-up work, and operational impact.

Use one file per durable decision so changes can be reviewed and superseded
cleanly.
