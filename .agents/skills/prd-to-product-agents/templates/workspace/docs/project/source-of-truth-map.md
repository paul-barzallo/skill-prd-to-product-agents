
# Source of Truth Map

This document maps every canonical artifact to its location, schema, primary steward, allowed mutation path, and consumers.

> **Rule:** If any two sources disagree, the canonical file listed here wins. Create a finding for the discrepancy.

## Operational state files

These files drive workflow decisions. They are mutated **only** through `prdtp-agents-functions-cli state` subcommands - never by direct YAML edits.

| Artifact | Canonical location | Schema | Primary steward | Mutation path | Consumers (prompts / agents) |
| --- | --- | --- | --- | --- | --- |
| Handoffs | `docs/project/handoffs.yaml` | `schemas/handoffs.schema.yaml` | pm-orchestrator | `prdtp-agents-functions-cli state handoff create`, `prdtp-agents-functions-cli state handoff update` | bootstrap-from-prd, client-review-uat, security-check, post-release-monitoring, release-readiness, daily-standup |
| Findings | `docs/project/findings.yaml` | `schemas/findings.schema.yaml` | qa-lead | `prdtp-agents-functions-cli state finding create`, `prdtp-agents-functions-cli state finding update` | security-check, post-release-monitoring, release-readiness, client-review-uat, daily-standup |
| Releases | `docs/project/releases.yaml` | `schemas/releases.schema.yaml` | devops-release-engineer | `prdtp-agents-functions-cli state release create`, `prdtp-agents-functions-cli state release update` | release-readiness, post-release-monitoring, client-review-uat |

## Governance files

These files are immutable after bootstrap (listed in `.github/immutable-files.txt`), except where noted.

| Artifact | Canonical location | Purpose | Immutability |
| --- | --- | --- | --- |
| Workspace instructions | `.github/copilot-instructions.md` | Global rules, memory contracts, execution workflow | Immutable (in manifest) |
| Agent instructions | `.github/instructions/agents.instructions.md` | Assembly model, execute boundaries, per-role rules | Immutable (in manifest) |
| Doc instructions | `.github/instructions/docs.instructions.md` | Ownership table, mutation rules | Immutable (in manifest) |
| Agent identities | `.github/agents/identity/*.md` | Immutable personality and behavior | Immutable (in manifest) |
| Agent model policy | `.github/agent-model-policy.yaml` | Model routing for all agents | Immutable (in manifest) |
| GitHub governance | `.github/github-governance.yaml` | Readiness states, reviewers, labels | Bootstrapped as a local skeleton; completed by `prdtp-agents-functions-cli governance configure` |
| Workspace capabilities | `.github/workspace-capabilities.yaml` | Runtime capability decisions for Git, gh, SQLite, markdownlint and reporting; operational gate only | Mutable via `prdtp-agents-functions-cli capabilities detect` / `capabilities authorize` |
| Operational state | `docs/project/{handoffs,findings,releases}.yaml` | Machine-managed canonical workflow state files | Protected (in manifest); mutated only via `prdtp-agents-functions-cli state` subcommands |
